### 关键澄清
1. API Gateway（不需要开发）
- 现成工具：Kong、Traefik、Nginx、Envoy
- 直接使用，无需开发
- 负责：路由、SSL、认证、限流、监控
2. API Service（可选，根据需求决定）
有两种实现方式：
#### 方式 A：不需要开发新服务（推荐）
- Controller 添加 HTTP Server（如 Gin）
- API Gateway 直接代理到 Controller
- 架构简单，维护成本低
#### 方式 B：需要开发独立服务
- 开发独立的 API Service
- 支持复杂业务逻辑、WebSocket 等
- 架构更复杂，但功能更完整
### 推荐方案
对于大多数场景，推荐方式 A：
```
前端 → API Gateway → Controller (添加 HTTP 接口) → Kubernetes API
```
优点：
- 不需要开发新服务
- 架构简单
- 快速上线
- 维护成本低
